All Jobs — filter tabs layout options

Five takes on how to lay out the service tabs + view filters on the All Jobs page. Click any pill to see the active state. None of these are wired up to real data — visual only.

Option A — Current state LIVE NOW
Two pill rows side by side. No labels. Shared mutually-exclusive state — clicking one row clears the other.
ProsVisually groups services together. Single state — easy to implement.
ConsTwo adjacent pill rows imply they combine (AND), but they don't. No labels explain the difference.
Option B — Labeled groups
Same structure as A but with tiny uppercase captions above each pill row.
Service
Views
ProsTiny change — just two text labels. Tells the user the rows are different concepts. Lowest-risk improvement.
ConsStill doesn't fix that the three "view" tabs are different concepts mashed together. Still mutually-exclusive when it doesn't look it.
Option C — Checkbox + "More views" dropdown MY PICK
Cancelled becomes an orthogonal checkbox that combines with services. Bins Out and Recurring move to a dropdown next to the search bar.
ProsClean — service row only has services. Cancelled can combine with any service (e.g. "cancelled Bin Rentals only"). Dropdown hides the rarely-used Bins Out/Recurring until needed.
ConsLargest behavior change. Cancelled jobs now show inline (with a faded style) rather than in their own section — has to be wired up.
Option D — One row with a visible divider
Revert to one pill row (pre-split state) but add a vertical divider between services and views.
ProsSmallest visual change. The divider hints "these belong to a different group" without changing state semantics.
ConsDoesn't really separate the concepts — they still look like part of one list. A divider is a weak signal.
Option E — Move special views into the sidebar
All Jobs row keeps only the real services. Bins Out and Recurring become their own sidebar pages (like Live Jobs). Cancelled stays as a checkbox toggle.

ProsCleanest separation — Bins Out and Recurring get treated as the views they actually are. Reaches them from anywhere via the sidebar, not just from inside All Jobs.
ConsMost disruptive — moves nav around. Users who relied on the current placement need to relearn where things live.
My recommendation: Option C. It fixes the conceptual mismatch (cancelled is orthogonal, not a service), keeps the row focused on actual services, and stashes the rare views in a dropdown. Option B is the safer "almost no-op" alternative if you don't want the behavior change.
